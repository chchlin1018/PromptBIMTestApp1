# 互動式修改測試場景

> 測試 Modifier Agent 的即時修改能力
> 每個場景包含: 初始 prompt → 一系列修改指令 → 預期系統反應

---

## 場景 A: 辦公大樓樓層調整

```
初始: "在 400 坪商三地上蓋 12 層辦公大樓"

修改 1: "請修改 12 樓層改為 9 個樓層"
  預期: 面積↓25%, 成本↓25%, 工期↓120天, 高度合規改善

修改 2: "層高改為 3.2 米"
  預期: 高度重算, MEP 管線高度調整

修改 3: "頂樓改為員工餐廳和露台"
  預期: 平面變更, 廚房 MEP 加入, 結構載重調整

修改 4: "加一層地下停車場"
  預期: 基礎加深, 車道坡道, 排煙系統, 成本↑15%
```

## 場景 B: 住宅規格微調

```
初始: "3 層透天住宅，4 房 2 廳"

修改 1: "主臥加一間更衣室"
  預期: 2F 平面調整, 可能壓縮次臥面積

修改 2: "一樓加車庫可停 2 台車"
  預期: 1F 平面重配, 結構柱位調整

修改 3: "改用鋼構"
  預期: 結構全部重算, 成本變化, 工期可能縮短

修改 4: "加設太陽能板"
  預期: 屋頂載重, 電力系統加入逆變器, 成本↑
```

## 場景 C: 數據中心設備調整

```
初始: "100MW 數據中心"

修改 1: "改為 50MW"
  預期: 規模減半, UPS/冷卻/配電全部縮小, 成本↓40%

修改 2: "冷卻改用液冷"
  預期: 取消冷卻塔, 加入 CDU, 管路系統變更

修改 3: "加設柴油發電機 N+1 備援"
  預期: 發電機室擴大, 油槽加入, 消防升級
```

## 場景 D: 連續微調驗證

```
初始: "5 層辦公樓"

修改 1: "退縮增加到 8 米" → 可建面積縮小
修改 2: "走廊加寬到 2 米" → 辦公面積縮小
修改 3: "加一部電梯" → 核心筒擴大
修改 4: "外牆改帷幕牆" → 成本↑, 外觀變更
修改 5: "撤銷上一步" → 恢復到修改 3 的版本
修改 6: "顯示版本 1 和現在的比較" → 差異報表
```
